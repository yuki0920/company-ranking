## Repository shape

Monorepo with three runtimes plus an OpenAPI contract that ties them together:

- `go/` — public read API (Chi-style `http.ServeMux` + handlers generated by `oapi-codegen` into `go/server/server.gen.go`). Models under `go/models/` are generated from the live DB with `xo` (`*.xo.go`); hand-written extensions sit alongside as `*_extend.go`. Entry point is `go/cmd/server`.
- `ruby/` — Rails app used **only** as a batch/data-processing pipeline. There are no controllers exposing HTTP; work is done via rake tasks under `ruby/lib/tasks/` (`save_securities`, `save_document_summary`, `save_document_detail`, `adjust_salary`, `destroy_old_documents`, `update_industry_category`). The default container command is a no-op (`compose.yaml` runs `echo rails api is not used.`).
- `typescript/` — Next.js 15 App Router frontend (`typescript/app/[lang]/...` for `en`/`ja` locales). The API client in `typescript/client/` is generated from the OpenAPI schema via `openapi-generator-cli` (typescript-fetch) — do not hand-edit it.
- `openapi/openapi.yaml` is the source of truth shared between Go server and TS client. After editing it, regenerate both sides (see commands below) and re-lint.

Data flow: Ruby rake tasks scrape JPX + call EDINET (`書類一覧API` → `書類取得API`) and write to Postgres (`securities`, `documents`, `industries`, `markets`). Go reads from the same Postgres and serves the Next.js frontend.

## Common commands

The top-level `Makefile` wraps each language's tooling in `docker compose run --rm <service>`, so prefer the Make targets over running tools on the host.

```sh
# Bootstrap (first time)
cp .env_sample .env && cp .envrc_sample .envrc
docker compose build
docker compose run --rm ruby bundle
docker compose run --rm ruby bundle exec rails db:setup
make go/install/tools
cd typescript && npm install && cd -

# Run the full stack (api on $API_HOST_PORT, web on $FRONT_HOST_PORT, swagger on :8000)
docker compose up

# Tests
make go/test         # go test -race -shuffle=on with coverage to go/coverage.out; runs scripts/setup_db.sh first
make ruby/test       # bundle exec rspec
# Single Go test:    docker compose run --rm go go test -run TestName ./server/...
# Single RSpec file: docker compose run --rm ruby bundle exec rspec spec/path/to_spec.rb

# Lint / format
make go/lint         # golangci-lint (use go/lint/fix to auto-fix)
make ruby/lint       # rubocop (use ruby/lint/fix)
make typescript/lint # next lint
make typescript/format # prettier --write

# Code generation from OpenAPI (run BOTH after editing openapi/openapi.yaml)
make go/generate/server         # oapi-codegen → go/server/server.gen.go
make typescript/generate/client # openapi-generator-cli → typescript/client/

# Generate Go DB models from a live database (requires DATABASE_URL)
make go/generate/models  # xo schema → go/models/*.xo.go (do not edit *.xo.go by hand)
```

`typescript/generate/client` runs `openapi-generator-cli` directly on the host, so it must be installed globally (`npm i -g @openapitools/openapi-generator-cli`). All other Make targets run inside Docker.

## Things to know before editing

- **Generated files**: `go/server/server.gen.go`, `go/models/*.xo.go`, and `typescript/client/**` are regenerated from `openapi/openapi.yaml` or the DB. Edit the schema (or the `*_extend.go` companion files) instead — never the `.gen.go` / `.xo.go` files.
- **Schema changes are a 3-step task**: edit `openapi/openapi.yaml` → `make go/generate/server` → `make typescript/generate/client`, then update the Go handlers in `go/server/server.go` and any frontend callers, and re-run the linters.
- **Ruby is batch-only**. If a feature seems to require new HTTP endpoints, add them to the Go server, not Rails.
- **Ports / env**: `API_HOST_PORT`, `FRONT_HOST_PORT`, `POSTGRES_PORT`, `DATABASE_URL`, `NEXT_PUBLIC_API_URL`, `EDINET_API_KEY`, `SLACK_WEBHOOK_URL` all come from `.env` and are referenced by `compose.yaml`. Swagger UI is hardcoded to `:8000`.
- **TypeScript has no unit tests** — verify changes with `npm run build` (or `docker compose run --rm typescript npm run build`) and `make typescript/lint`.
- **External services**: this codebase already integrates with EDINET, JPX scraping, Slack webhooks, GCP (Cloud Build / Terraform), and Supabase. Adding a *new* third-party service may require an internal "External Service Review" — flag it to the user first.
